1. Variables


A variable is a place in the computer's memory where some kind of value sometimes a string (some letters), sometimes a number are stored. Like the value 5 or the string "hello”. A variable can have a simple name like just the letter x or any combination of letters like last_name or social_security_number, but not spaces, last name is not a valid variable name. If we have a variable called social_security_number we could assign it the value 123456789 as a sample social security number. if you wanted to dashes maybe you would want to use a string like "123-45-6789". Strings are surrounded by double quotes, numbers are just written bare. Variables as their name implied can vary, which seems to be different to the mathematical view on variables, in math if you say x=5 then x is always equal to 5, but in programming we can say x=3 and then x=5 and the value would first be set to 3 and then change to 5(These are called assignments in programming). In the case of a string, the value between the double quotes will be assigned to the variable, for example if we had last_name we could assign "Einstein” to it easily as follows: last_name = "Einstein”.

this may be the most difficult part to explain of programming, almost everything you see in a computer program is a statement. Lets talk about assignment statements. For example, lets take the following program

We have the x on the left of the equal sign called an "lvalue” (the variable to the left of the equal sign) which is the same as the first x=3, where the x is what stores the value, then we have x+1, the value of x at this point of the program is 5 because it comes after x=5 so when we do x+1 it is the same as doing 5+1, and then finally we assign that to the lvalue, so at the end of the program, x will contain 6 which is 5+1. Other kinds of statements we will look at later in this text are things like function calls, return statements and conditional statements.

The following makes a pyramid, it is an example of how to use several loop bodies for a forloop in the code
lines 0005-0009:
This defines a class(something like a cookie cutter for variables called "objects") to initialize global variables. It sets "output" to an empty string (used to build each row of the pyramid) and "ch" to "*" (the character for the pyramid, like stars in a Christmas tree).
In a class def init(self) is called a "constructor" and it is a function when you cut a cookie(create an object) form the cookie cutter.
line 0010 "Function call g = initGlobals"
This creates an "instance of the class" (a cookie cut out of the cookie cutter) and assigns it to the g variable, making output and ch accessible throughout the script as g.output and g.ch
lines 0012 to 0014:
This defines a function (loop_body1) that adds a single space (' ') to the global output string. The parameter "n" is ignored—it's just a placeholder because the loopy lib's forloop passes a value to the body. This is used to add leading spaces for indenting each row of the pyramid.
lines 0015 to 0017
Similar to above, but adds the current character (globals.ch, which is '*') to the output. Again, "n" is ignored. This builds the asterisk part of each row.
lined 0018 to 0019
This defines a condition function that checks if n1 equals n2 using the loopy lib's conditional and op functions. This will be used as the stopping condition for the for loops (stop when the loop variable equals the max value).
line 0020:
This sets the global condition in the loopy lib to the equality check (==). By default, the lib might use >, but here we change it to == so the loop includes the max value and stops when it hits it.
lines 0022-0034:
This defines the function that prints one row of the pyramid. Parameters:
ch: The character to use (e.g., '*').
times: The row number (1 to 5), which controls how many asterisks to print.
triangle_width: Fixed at 5 here, controls the overall size of the pyramid.
line 0023:
Resets the output string for a new row.

0028:Runs a "for loop" from 0 to repeat_end (inclusive, because condition is ==). For each iteration, it calls loop_body1, adding a space each time. For repeat_end=4, it adds 5 spaces (iterations: 0,1,2,3,4).
